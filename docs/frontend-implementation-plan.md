# 잇장 프런트엔드 구현 계획

## 1. 기술 방향

초기 MVP는 React, TypeScript, Vite 기반의 모바일 우선 웹앱으로 구현한다.
백엔드 API가 준비되기 전까지는 mock data를 사용하고, 이후 API service 계층만 교체한다.

## 2. 디자인 방향

- 초록색 중심의 빈티지한 옷장 무드
- 나무, 패브릭, 종이 질감이 떠오르는 따뜻한 색감
- 추천 결과에서는 내 옷장 보유 아이템을 가장 먼저 노출
- 보유 아이템에는 테두리와 안내 배지를 표시

## 3. 화면 구성

### 홈 화면

- 잇장 로고와 서비스명 표시
- 무신사 로그인, 카카오 로그인 진입 버튼 표시
- 살짝 열린 옷장 사이로 초록색이 보이는 첫 화면 연출

### 사용자 정보 입력 화면

- 사용자 기본 정보 컴포넌트와 선택 정보 컴포넌트 분리
- 키, 몸무게, 발 사이즈, 선호 핏, 선호 스타일, 기피 스타일 입력

### 내 옷장 조회 화면

- 상단 카테고리 탭 제공
- 상의, 하의, 아우터, 신발, 가방, 악세서리 기준 조회
- 사진 카드 그리드로 옷 목록 표시

### 옷장 등록 화면

- 직접 등록 탭
- 구매내역 가져오기 탭
- 구매내역 가져오기 대상: 무신사, 29CM, 네이버쇼핑
- 초기에는 캡처 업로드 또는 URL 입력 흐름으로 설계

### 옷 추천 선택 화면

- 최상단 프롬프트 입력창과 전송 버튼
- 상황, 분위기, 목적 태그 다중 선택
- 예시 태그: 데이트, 심플, 러블리, 비오는날, 등교, 면접

### 추천 결과 화면

- 상의, 하의, 신발, 아우터, 가방 등 카테고리별 추천 후보 표시
- 내 옷장 아이템 우선 노출
- 내 옷장 보유 아이템에는 "내 옷장에 있는 옷이에요!" 표시

### 커뮤니티 화면

- 당근마켓처럼 카테고리별 중고 판매 및 구매 피드 제공
- 옷 카테고리, 가격, 상태, 거래 상태를 표시할 수 있는 구조로 설계

## 4. 추천 구현 순서

1. 프런트 프로젝트 생성 및 라우팅 설정
2. 디자인 토큰과 공통 레이아웃 구성
3. mock data와 타입 정의
4. 홈 화면 구현
5. 사용자 정보 입력 화면 구현
6. 옷장 조회 및 등록 화면 구현
7. 추천 선택 화면 구현
8. 추천 결과 화면 구현
9. 커뮤니티 화면 구현
10. 반응형 및 모바일 사용성 점검
